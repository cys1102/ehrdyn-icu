# KDD-RV successor clinical task review

Review one task independently against its machine-readable config, immutable
task-table row, task card, and aggregate construction evidence.

- Task identifier and version:
- Reviewer specialty:
- Relevant clinical role and experience:
- Anchor identity is clinically defensible: yes / no / changes required
- Inclusion and exclusion rules are clinically defensible: yes / no / changes required
- Recorded action representation is interpretable as exposure: yes / no / changes required
- Timing and stay-containment contract is defensible: yes / no / changes required
- Prohibited-feature and leakage rules are sufficient: yes / no / changes required
- Outcome and censoring description is defensible: yes / no / changes required
- Wording avoids treatment, causal, policy, utility, and deployment overclaims: yes / no / changes required
- Decision: approve / approve with changes / reject
- Required changes:
- Optional comments:
- Review date:

Do not include patient examples, identifiers, exact event times, raw notes, or
other protected/restricted information.
