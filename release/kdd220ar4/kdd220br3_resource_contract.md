# KDD220BR3 resource and execution contract

KDD220BR3 is authorized only from the exact pushed KDD220AR4 commit when
KDD220AR4 decides
`complete_respiratory_missing_action_filter_repair_ready_for_kdd220br3`.
KDD220AR4 itself is code-only and provides no credentialed resource result.

The future credentialed run must use the public default chunk size of 250,000
rows and an audited process monitor such as `/usr/bin/time -v`. It must report
wall time, user and system time, maximum resident set size, input and output
bytes, temporary-disk use, chunks processed, maximum retained rows per chunk,
and compression encoding for every documented high-volume table. The hard
peak-memory gate remains 64 GiB and the design target remains 32 GiB. Crossing
32 GiB without crossing 64 GiB is a named accessibility limitation; crossing
64 GiB stops before parity references are opened.

The constructor must run from a fresh clone with official MIMIC-IV v3.1 flat
files, the frozen public configuration, and a read-only local root argument.
It must freeze source, command, schemas, output locations, tolerances, and
chunk size before data access, finish constructing all five task interfaces
before opening references, and preserve aggregate diagnostics on controlled
runtime stops. No KDD220BR3 execution occurs in this stage.
