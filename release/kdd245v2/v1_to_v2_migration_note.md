# v1.3.0 to proposed v2.0.0 migration

KDD220M1 freezes a candidate canonical-v2 task definition and reports its
historical differences as migration provenance, not parity. KDD220M2 was meant
to refresh every affected paper-facing real-EHR result, but the public
constructor stopped in the sepsis suspected-infection blood-culture path before
interface freeze or training. Consequently no v2.0.0 release exists and v1.3.0
results must not be relabeled, pooled, or treated as refreshed v2 evidence.
